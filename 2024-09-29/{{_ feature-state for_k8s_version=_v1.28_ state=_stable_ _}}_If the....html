Kubernetes v1.28에서는 ProxyTerminatingEndpoints 기능이 안정화(stable) 단계에 도달했습니다. 이 기능은 종료 중인 엔드포인트로의 트래픽 처리를 개선하여 서비스의 안정성과 가용성을 향상시킵니다.

주요 특징

  • kube-proxy의 동작 변경: ProxyTerminatingEndpoints 기능이 활성화되고 트래픽 정책이 Local로 설정된 경우, kube-proxy는 서비스의 엔드포인트 선택에 더 복잡한 알고리즘을 사용합니다.
  • 로컬 엔드포인트 처리: kube-proxy는 노드에 로컬 엔드포인트가 있는지, 그리고 모든 로컬 엔드포인트가 종료 중인지 확인합니다.
  • 종료 중인 엔드포인트로의 트래픽 전달: 모든 로컬 엔드포인트가 종료 중이라면, kube-proxy는 이러한 종료 중인 엔드포인트로 트래픽을 전달합니다.
  • 비종료 엔드포인트 우선: 그렇지 않은 경우, kube-proxy는 항상 종료되지 않은 엔드포인트로 트래픽을 전달하는 것을 선호합니다.

목적 및 이점

  1. 연결 드레이닝: NodePort와 LoadBalancer 서비스가 externalTrafficPolicy: Local을 사용할 때 연결을 원활하게 드레인할 수 있게 합니다.
  2. 롤링 업데이트 중 안정성: 디플로이먼트의 롤링 업데이트 과정에서 로드 밸런서 뒤의 노드가 N개에서 0개의 레플리카로 전환될 때 발생할 수 있는 문제를 해결합니다.
  3. 트래픽 손실 방지: 외부 로드 밸런서가 헬스 체크 프로브 사이에 0 레플리카 노드로 트래픽을 보낼 수 있는 상황에서도 트래픽 손실을 방지합니다.
  4. 우아한 종료: 종료 중인 엔드포인트로 트래픽을 라우팅함으로써, 스케일 다운 중인 노드의 Pod들이 트래픽을 원활하게 받고 드레인할 수 있게 합니다.
  5. 로드 밸런서 동기화: Pod 종료가 완료될 때까지 외부 로드 밸런서는 노드의 헬스 체크 실패를 감지하고 백엔드 풀에서 해당 노드를 완전히 제거할 수 있습니다.
이 기능은 특히 externalTrafficPolicy: Local을 사용하는 서비스의 안정성을 크게 향상시키며, 롤링 업데이트 중 트래픽 손실을 최소화하는 데 중요한 역할을 합니다.
